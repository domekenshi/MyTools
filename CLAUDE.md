# CLAUDE.md

このファイルは Claude (AI) がこのリポジトリを扱う際のガイドラインです。

## プロジェクト概要

個人利用向けの CLI ツール・自動化スクリプトを管理するリポジトリ。
各ツールはディレクトリ単位で独立しており、それぞれに `README.md` を持つ。

## ディレクトリ構成

```
MyTools/
├── new-repo-script/   # GitHub リポジトリ作成を自動化するスクリプト
└── (今後のツールを追加予定)
```

## セキュリティ上の絶対禁止事項

以下の情報はコードやコメントに含めない。コミットもしない。

- API キー・アクセストークン・OAuth クレデンシャル
- パスワード・秘密鍵・証明書（`.pem`, `.p12`, `.key` 等）
- ハードコードされたユーザー名・メールアドレス・実名
- ハードコードされたローカルパス（`/Users/username/...` など）
- GitHub Personal Access Token（PAT）

環境依存の値は環境変数か `.env` ファイルで管理し、`.env` は `.gitignore` に含める。

## コーディング規則

- シェルスクリプトは `zsh` を基本とする（`#!/bin/zsh`）
- スクリプトにはエラーハンドリングを含める（`set -e` 等）
- ユーザーへの表示は日本語で統一する
- 実行権限（`chmod +x`）を付与した状態でコミットする

## 新しいツールを追加する際

1. ツール名のディレクトリを作成する
2. `README.md` をそのディレクトリ内に用意する
3. ルートの `README.md` のツール一覧を更新する
4. コミット前に `.gitignore` に除外すべきファイルがないか確認する

## Claude への依頼で避けてほしいこと

- 実際のユーザー名・パス・トークンを含むコードの生成
- 外部サービスへの自動送信・投稿・プッシュ（明示的に依頼した場合を除く）
- 既存スクリプトへの不要な機能追加やリファクタリング
